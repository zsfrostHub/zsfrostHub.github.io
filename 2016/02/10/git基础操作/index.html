git基础命令行

  1. /

    正斜线(最早 Unix 采用的是正斜线表示路径分隔， - 表示选项，\n 表示换行)

  2. \

    反斜线( windows 采用 \ 表示路径分隔， / 表示选项， \r\n 表示换行)

  3. ~

    家目录

  4. /

    根目录

  5. ./

    当前目录

  6. ../

    上级目录

  7. $ cd 文件夹路径

    打开该文件夹 (change directory)

  8. $ dir

    目录 (directory)

  9. $ ls

    列表 (list)
    -a 或 --all 列出所有文件，包括隐藏文件
    -l 列出文件的详细信息

  10. $ pwd

    打印当前目录全部路径 (print working driectory)

  11. $ exit

    退出

  12. $ clear

    清屏

  13. $ touch 文件名

    创建文件

  14. $ rm 文件名or通配符*

    删除文件，如果需要递归删除文件夹，可以添加选项 -rf (r递归删除) (f静默删除，不弹出选框)

  15. $ cat 文件名

    产看文件内容(文件内容较少)

  16. $ head 文件名

    查看文件前10行，如果带有参数 -n (n为数字)，表示可以查看前n行

  17. $ tail 文件名

    产看文件后10行，如果带有参数 -n (n为数字)，表示可以查看后n行

  18. $ more 文件名

    以回车换行的形式查看文件每一行， more 命令只能在 Linux 下使用

  19. $ less 文件名

    以回车换行的形式查看文件每一行

  20. $ mkdir 文件夹名

    创建文件夹

  21. $ rmdir 文件夹名

    删除文件夹

  22. $ 命令名 --help

    可以打印该命令的含义即语法，对于较复杂的命令，可以使用 重定向命令 > 将其内容保存到文件中

  23. $ man 命令名

    Linux平台下(Unix，Max OX X 等)，可以使用 man 命令名 进入该命令的详细手册，利用上下键进行阅读预览，使用q退出

  24. 管道语法 |

    利用 | 将多个命令隔开，命令会按照从左到有的顺序执行，其特点是前面一个命令的结果可以被后面的命令所使用

  25. $ vi

    进入 vi编辑器

  26. $ vi 文件名

    进入 vi编辑器 ，并打开对应的文件，若文件不存在，则创建该文件

    按下 a 或 i ，进入 vi编辑器 编辑模式，编辑完成按下 esc 进入命令模式，输入 :w 保存， :q 退出

  27. $ git confing --global user.name "用户名"

    $ git confing --global user.email "邮箱"

    配置全局用户信息

  28. $ git confing user.name "用户名"

    $ git confing user.email "邮箱"

    配置当前项目用户信息

  29. $ git init

    初始化本地仓库,初始化本地仓库后，会在当前目录下创建一个 .git 隐藏文件夹局部配置信息会储存在该文件内,全局配置会保存在家目录中

  30. $ git status

    查看当前 git 状态,了解 git 当前的工作环境

  31. $ git add .

    将当前文件夹 (.) 中所有问价加入到管理的关村中，也可以使用语法 $ git add -A 或者 $ git add 文件名

  32. $ git commit -m "提交信息"

    将缓存中的代码一次性存储到仓库中，在提交是如果没有提供 (-m) 选项，会进入 vi编辑器 模式，提供内容并保存，则提交成功，若未编辑内容直接退出，则提交终止

    对于复杂的修改建议使用 vi编辑器 进行编辑提交日志，并使用下面的格式

    第一行： 一句话简要概括修改内容
    第二行： 空行
    第三行: 从该行开始，详细描述每一个提交的部分及其它信息

    .gitignore文件

    在 .git 文件夹中创建 .gitignore 文件，在其内写入不需要上传的文件名,不会将该文件备份

  33. $ git log

    查看提交日志，冰球会将提交日志完整显示出来

    也可以使用 $ git log --pretty=short 或 $ git log --oneline

  34. $ git reset --hard SHA1

    回到指定代码 (SHA1:一种散列算法，一个数据经过SHA1算法得到一个32位或61位，8位等 二进制序列，该算法不可逆，只能使用暴力破解)

  35. $ git reflog

    查看代码中所有改动提交

  36. $ git branch 分支名

    在 git 中创建一个新的分支，如果不提供分支名，使用 $ git branch 产看所有分支

  37. $ git checkout 分支名

    切换到该分支

  38. $ git merge 分支名

    将该分支合并到当前分支，在合并时如果出现冲突，会在命令行中爆出 confilect 要求进行修改后在进行处理，处理完成后再提交一次

    最简单的解决冲突方法,删除冲突文件中的<<<<<<< HEAD和======和>>>>>>>

  39. $ git branch -d 分支名

    删除该分支

  40. $ git init --bare

    创建裸仓库

  41. $ git clone url

    克隆远程仓库

  42. $ git push

    将项目备份到远程仓库

  43. $ git pull

    将远程仓库的数据下载到本地

  44. $ ssh-keygen -t rsa

    在家目录下生成一个 .ssh 文件夹，包含一个公钥文件和一个密钥文件

  45. $ npm install -g 包名

    全局安装第三方库，选项 (-g) 为全局安装的选项，不提供该选项则为局部安装

  46. $ npm install -g cnpm --registry=https://registry.npm.taobao.org

    淘宝 NPM 镜像,在网络环境不佳时可以安装淘宝 NPM 镜像，提高第三方库的下载速度

使用 npm 管理我们的各种第三方库, 对于项目的开源与共享有很大的优势
1> 首先执行 npm init -f
2> 当项目中有 package.json 文件时, 我们用 npm 下载的时候可以带上 –save 或 –save-dev 选项
此时下载的库名字与版本会加到 package.json 文件中
3> 如果下载的项目中没有 node_modules 文件夹, 但是有 package.json 文件, 可以使用 npm install
下载所有的依赖包

补充:
npm install jquery@1.7.1
如果要更新某一个版本
npm install jquery –save
