服务器类型

  1. 按服务类型:文件服务器、数据库服务器、邮件服务器、Web 服务器等
  2. 按操作系统:Linux服务器、Windows服务器等
  3. 按应用软件: Apache服务器、Nginx 服务器、IIS服务器、Tomcat服务器、Node服务器等

服务器软件

  1. 文件服务器:Server-U、FileZilla、VsFTP等（FTP是File Transfer Protocol文件传输协议）；
  2. 数据库服务器:oracle、mysql、SQL server、DB2、ACCESS等；
  3. 邮件服务器:Postfix、Sendmail等；
  4. HTTP 服务器:Apache、Nginx、IIS、Tomcat、NodeJS等
  5. 常见的运行在服务端的编程语言包括 PHP、Jsp、Asp、Python、Ruby、Perl等

表单提交

通过form表单把数据提交到后台,表单提交会发生页面的跳转,如果后台运算数据较慢 浏览器要一直等待,推荐使用ajax提交方法

  1. get
    1. 前端:
      1. 提交的地址 在form标签上写action属性 值代表要把input中的值传给哪个程序
      2. 数据的保存 在input的标签上写name属性的值
      3. 提交方式　　在form标签上写method属性 现在有两种提交 get(默认的提交方式) post
    2. 后端:
      1. 接受前端数据 用一个数组接受 先用$_GET数组(php中早就定义好的 内置数组) 用什么样的方式提交就用什么数组接受
    3. 特点:
      1. 提交的内容在地址栏中 格式:提交地址?提交内容(key=value&key=value) url编码:以%开头后面跟两个十六进制的数字
      2. 不安全
      3. 提交的内容容量比较小
      4. get代表从服务器获取内容
      5. get方式请求数据较快
  2. post
    1. 前端:
      1. 提交的地址 在form标签上写action属性 值代表要把input中的值传给哪个地址
      2. 数据的保存 在input的标签上写name属性
      3. 提交方式　　在form标签上写method属性 必须显式声明提交方式post
    2. 后端:
      1. 接受前端数据 用一个数组接受 只能用$_POST 用什么样的方式提交就用什么数
    3. 特点:
      1. 提交的内容在请求正文(请求体)中 格式: key=value&key=value;
      2. 相对get安全
      3. 提交的内容的容量 理论上 无限大的 受限于服务器和浏览器
      4. post发送数据 把前端的数据发送到后台去
      5. post方式提交数据较慢
      6. post请求有请求体

http协议两部份

  1. 请求
    1. 请求行 请求方式 请求地址
    2. 请求头 key:value
    3. 请求正文 key=value&key1=value1
  2. 响应
    1. 响应行 协议及版本 状态码 状态码描述
    2. 响应头
    3. 响应正文

请求报文:

1.  GET方式:

    请求行

        GET /day02-http&ajax/4-code/02-http/userLogin.php?  username=wanlum&  password=123456 HTTP/1.1

    请求头

        Host: localhost

        Connection: keep-alive

        Upgrade-Insecure-Requests: 1

        User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)   AppleWebKit/537.36 (KHTML, like Gecko)Chrome/59.0.3071.115Safari/537.36

        Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8

        Referer: http://localhost/day02-http&ajax/4-code/02-http/userLogin.html

        Accept-Encoding: gzip, deflate, br

        Accept-Language: zh-CN,zh;q=0.8

2.  POST方式:

    请求行 

        POST /day02-http&ajax/4-code/02-http/userLogin.php HTTP/1.1
        格式:请求方式 请求地址  协议名称/版本

    请求头

        Host: localhost
        代表主机名称

        Connection: keep-alive
        代表是否开启长链接 

        Content-Length: 31
        代表请求内容的长度(中文一个占三个字节)

        Cache-Control: max-age=0
        是否开启缓存     开启缓存 但是立即删除

        Origin: http://localhost
        域名的源

        Upgrade-Insecure-Requests: 1
        安全请求

        User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)AppleWebKit/537.36 (KHTML, like Gecko)Chrome/59.0.3071.115Safari/537.36
        用户代理

        Content-Type: application/x-www-form-urlencoded
        告诉服务器请求内容是表单的url的编码 

        image/png image/jpg  audio/mp3
        浏览器告诉服务器我能接受的内容的类型 

        Accept: text/html,application/xhtml+xml,application/xml;q=0.9, image/webp,image/apng,*/*;q=0.8
        我们把这样的格式大类型/具体类型叫做MIME类型 作用是告诉对方用什么样的方式去解析内容

        Referer: http://localhost/day02-http&ajax/4-code/02-http/userLogin.html
        当前的页面是从那个页面跳转过来的

        Accept-Encoding: gzip, deflate, 
        浏览器告诉服务器能接受的内容的压缩格式 

        Accept-Language: zh-CN,zh;q=0.8
        浏览器告诉服务器我所能接受的语言

    请求正文

        username=wanlum&password=123456

        text/html text/css text/javascript  
        叫做MIME类型 作用:告诉浏览器你如何去解析我标签中的内容

响应报文:

响应行

    HTTP/1.1 200 OK
    格式:协议/版本号 状态码  状态码描述

响应头

    Date: Fri, 07 Jul 2017 06:44:11 GMT

    Server: Apache/2.2.21 (Win32) PHP/5.3.10

    网站用什么技术开发的
    X-Powered-By: PHP/5.3.10

    响应内容的长度
    Content-Length: 60

    长连接的时间  5s以后算超时   超时之后 开始尝试链接 尝试一次max-1  直到100被减完了 就不再尝试了
    Keep-Alive: timeout=5, max=100

    Connection: Keep-Alive

    服务器告诉浏览器我给你的内容的格式
    Content-Type: text/html

响应正文

    Array ( [username] => wanlum [password] => 123456 )
